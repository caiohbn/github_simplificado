Git básico + Pull Request

Material de apoio para uma mini‑palestra. Vamos praticar commit, checkout, pull, push e abrir um Pull Request.

Objetivo: cada participante fará um PR editando participants.md neste repositório.

1) Conceitos rápidos

  • commit: salva um “pacote” de alterações no histórico local.
  • checkout: muda de branch ou cria uma nova (git checkout -b).
  • pull: baixa as alterações do remoto e integra na sua branch atual.
  • push: envia seus commits locais para o repositório remoto.

2) Pré‑requisitos

  • Conta no GitHub e Git instalado (git --version).
  • Editor de texto (VS Code, etc.).
  • Faça fork deste repositório e depois clone o seu fork.

3) Fluxo da atividade (PR)

  1. Crie sua branch de trabalho: 
    git checkout -b feat/seu-nome
  2. Edite o arquivo participants.md e adicione uma linha com seu nome e cidade.
    Se o arquivo não existir, crie um novo na raiz do repositório com o conteúdo do exemplo mais abaixo.
  3. Veja o que mudou: 
    git status
  4. Adicione e faça commit das alterações: 
    git add participants.md
git commit -m "chore: adiciona <seu-nome> ao participants.md"
  5. Sincronize sua branch com o remoto (se necessário): 
    git pull --rebase origin main
  6. Envie sua branch para o seu fork: 
    git push -u origin feat/seu-nome
  7. Abra um Pull Request no GitHub do seu fork para o repositório original (base: main).

4) Comandos úteis

# configurar usuário (uma vez)
git config --global user.name "Seu Nome"
git config --global user.email "voce@exemplo.com"

# ver histórico
git log --oneline --graph --decorate --all

# alternar de branch
git checkout main

# atualizar sua cópia local
git pull origin main

# enviar commits
git push origin <sua-branch>

5) Dicas rápidas

  • Commits pequenos e mensagens claras.
  • Uma feature por branch.
  • Antes do push, rode git pull --rebase para evitar conflitos.
  • Se der conflito, abra o arquivo, resolva, depois git add e git rebase --continue.

6) Exemplo de participants.md

Crie um arquivo na raiz chamado participants.md e adicione seu nome como no exemplo:

# Participants

- Seu Nome – Sua Cidade
- Ada Lovelace – Londres
- Grace Hopper – Nova Iorque

✅ Dica: use acentos normalmente; o Markdown suporta caracteres UTF‑8.

7) Roteiro da mini‑palestra (facilitador)

  1. Apresentar o fluxo Git e o objetivo (5 min).
  2. Mostrar como fazer fork e clone (5 min).
  3. Criar branch com checkout -b, editar participants.md, commit, push e abrir o PR (10–15 min).
  4. Encerrar revisando PRs e fazendo merge ao vivo (5 min).

8) Troubleshooting rápido

"Permission denied" ao dar git push

Verifique se o remoto aponta para o seu fork (origin). Use git remote -v. Se estiver errado: git remote set-url origin <url-do-seu-fork>.

Confli tos ao dar pull --rebase

Abra os arquivos marcados, resolva os conflitos, rode git add . e em seguida git rebase --continue.

Esqueci de criar branch antes de editar

Crie agora e mova o commit: git checkout -b feat/seu-nome e continue normalmente.
